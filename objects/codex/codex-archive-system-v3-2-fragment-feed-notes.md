v3.2 notes are now extended by [Codex Archive System v3.3 Modest Table Notes](/objects/codex-archive-system-v3-3-modest-table-notes).
Use v3.2 as fragment/feed context and v3.3 for current card-image staging and inspection behavior.

v3.2 is the feed-behavior release.

Operator value prop:
- Fragments now communicate at first sight; the object remains inspectable, but the main signal already lands in the feed.

Work chunks and wins:

1. Fragment-first feed communication
   Value prop: A fragment can publish as a complete unit of language instead of pretending to be a teaser for a later click.
   - Promote fragment body text to the primary feed surface.
   - Remove generic empty-preview treatment for feed fragments.
   - Keep canonical object pages available as optional context/metadata inspection.

2. Object-specific feed behavior
   Value prop: The system can honor different object types without flattening everything into one card template.
   - Give fragments a dedicated feed-interlude rendering path.
   - Let feed fragments size to natural content height instead of reserving generic card rectangles.
   - Move fragment identification to a quiet foot label so the language stays primary.
   - Treat fragments as first-class objects without changing schema or breaking object pages.

3. Prompt/return conversation preview
   Value prop: Operator/system exchanges can be previewed in the feed as readable contact, not collapsed stacks.
   - Detect speaker-style lines such as `Operator:` and `System:`.
   - Render exchange turns as `Prompt` and `Return`.
   - Preserve intentional breathing room between turns.
   - Widen conversational measure so system replies do not collapse into narrow vertical columns.

4. Presentation lesson now codified
   Value prop: Presentation specificity can increase while the archive spine stays stable.
   - The object model remains unchanged.
   - The viewer became more type-aware.
   - The feed now behaves more like an interpretive surface and less like a uniform index.

No schema expansion is proposed in v3.2.
No new object type is proposed in v3.2.
