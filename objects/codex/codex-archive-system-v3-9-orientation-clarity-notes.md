v3.9 extends [Codex Archive System v3.8 Specimen Surface Notes](/objects/codex-archive-system-v3-8-specimen-surface-notes).
Use v3.8 for specimen-grade object surfaces and v3.9 for current archive orientation, naming clarity, and human framing.

v3.9 is the orientation-clarity release.

Operator value prop:
- A first-time visitor now gets a clearer answer to what this archive is, what kind of work lives here, and where to go for the human context behind it.

Work chunks and wins:

1. Header identity now favors clarity over domain shorthand
   Value prop: Site chrome names the thing instead of making readers infer it from the URL.
   - Standardized the public header label to `Codex Archive`.
   - Kept the formal system name out of the main chrome so the site can introduce itself more directly.

2. The feed hero now opens with one readable invitation
   Value prop: The homepage explains the archive in plain language before the reader meets the denser system vocabulary.
   - Rebuilt the opening hierarchy around `Creative Work`, `A Living Archive`, and one combined explanatory line.
   - Explicitly named collage art alongside poems, notes, artifacts, and experiments.
   - Tuned the display title measure so it resolves as a single-line thesis on desktop.

3. Orientation now carries the deeper human context
   Value prop: The reason for the archive can live in its proper place without overloading the feed.
   - Reframed the Orientation hero as `Archaeological Mythology in Practice`.
   - Added a full-width personal note under the operator profile so survival, continuity, and lived experience can sit beside the system map.

4. Public copy now aims for empathy as well as precision
   Value prop: The archive introduces itself like a place a person can enter, not only a system they must decode.
   - Reduced doctrinal language in the feed and header surfaces.
   - Moved more of the abstract system explanation into Orientation, where readers can choose to go deeper.
   - Let the homepage lead with creative work, object language, and a clearer invitation path.

No schema migration is required for v3.9.
No new object type is introduced in v3.9.

Implementation note:
- This pass is less about new functionality than tone control. The archive now meets the reader with more clarity, more empathy, and less needless interpretive load.
